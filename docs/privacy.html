Legal & Privacy

KIDO™ Privacy Policy

Last updated: July 2026 Applies to: KIDO Android app (all versions) Contact: ssshekharsamal@gmail.com

Core commitment

KIDO is built to work entirely on your phone. Your child's conversations, memories, name, and preferences never leave your device. There is no account, no cloud sync, and no server that receives personal information.

01 — About this policy

Who this applies to

This policy covers the KIDO companion robot Android application published by Shekhar Samal. It describes what information the app handles, where that information lives, and what choices you have as a parent or guardian.

KIDO is intended for use by children under parental supervision. We treat all users as if they may be children and apply the same privacy standards across the board.

02 — What KIDO stores

Everything stays on your phone

KIDO stores the following information locally in your phone's private storage. Nothing in this list is transmitted, backed up to a cloud service, or visible to us.

What Where it lives Who can see it
Child's name & profile On-device database Device only
Conversations with KIDO On-device database Device only
Learned preferences & memories On-device database Device only
Face recognition data On-device storage Device only
Parent settings & PIN On-device encrypted storage Device only
Diagnostic logs (errors) On-device file Device only

03 — Internet connections

The only times KIDO contacts the internet

KIDO operates offline by design. There are two narrow situations where it makes an outbound network request, and neither involves personal information.

Situation Service contacted What is sent What is NOT sent
Child asks a factual question DuckDuckGo (safe search) & Wikipedia Network The question topic only Name, age, location, device ID, or any personal detail
Child asks about the weather wttr.in (no-account weather API) Network Your device's IP address (standard for any internet request) and, optionally, a city name if spoken Name, precise GPS location, device ID

Neither service receives an account identifier, device ID, or any information that could be linked back to your child. DuckDuckGo operates under its own strict no-tracking policy (safe search parameter kp=1 is always set). wttr.in is a privacy-respecting open-source weather service that does not build user profiles.

If you do not want either connection to occur, KIDO will still function fully: factual questions fall back to the on-device language model, and weather questions return a polite "I'm not sure" response.

04 — Model downloads

Downloading KIDO's brain on first setup

On first launch, KIDO downloads AI model files (speech recognition, voice, object detection, and language model) from GitHub and public model repositories. These downloads are one-time and can resume if interrupted.

During download, only the model file is fetched. No personal information, device identifiers, or usage data are sent. Once installed, these models run entirely offline.

Model files are stored in your phone's private app storage and are removed when you uninstall the app.

05 — What we never do

Hard limits, not policy choices

The following are architectural constraints, not just policy statements. The app does not contain the code to do any of these things.

  • Sell, share, or license your child's data to any third party
  • Display advertisements of any kind
  • Create a user account or require registration
  • Upload conversations, memories, or voice recordings to a server
  • Use analytics SDKs, crash-reporting services, or behavioural tracking
  • Access your device's contacts, calendar, SMS, or call history
  • Track your location via GPS
  • Send push notifications without an explicit on-device trigger
  • Run in the background when the app is closed

06 — Your rights & controls

You own everything KIDO stores

Because all data lives on your device, you have complete control without needing to contact us.

  • Export your family data From the Parent Dashboard → Family Archive → Export. Downloads a complete JSON file of all memories, profiles, and conversation history to your device.
  • Delete a family member's data From the Parent Dashboard → Profiles → select member → Remove. Permanently erases all memories, conversations, and face recognition data for that person.
  • Delete everything Uninstalling the app removes all stored data permanently. Android's standard app data clear (Settings → Apps → KIDO → Clear Data) also works.
  • Disable web lookup Settings → Knowledge → disable "Web answers" and "Weather". KIDO operates fully offline from that point.
  • View diagnostic logs Settings → Advanced → View Diagnostics. Logs contain only technical events (model load times, errors) and no conversation content.

07 — Children's privacy

COPPA and children under 13

KIDO is designed specifically for young children and complies with the U.S. Children's Online Privacy Protection Act (COPPA) and substantially equivalent regulations including the EU's GDPR-K provisions.

We do not knowingly collect personal information from children online. All data described in this policy is stored locally on the family's own device. No information is transmitted to us or to third parties in a form that could identify a child.

The Parent Dashboard is PIN-protected and is the only place where family profiles can be created, edited, or deleted. We encourage parents to review KIDO's settings before handing the device to a child.

If you believe a child's information has been handled improperly, please contact us at ssshekharsamal@gmail.com and we will respond within 72 hours.

08 — Camera & microphone

How KIDO uses device sensors

Microphone: Used only to hear spoken questions and commands while KIDO's listen window is active (triggered by wake word, screen tap, or play mode). Audio is processed entirely on-device by the speech recognition engine. No audio is ever recorded to a file or transmitted.

Camera: Used to detect faces (for profile recognition and presence detection) and objects (for play modes like Treasure Hunt). Camera frames are processed in memory at approximately 1–2 frames per second and immediately discarded. No images are saved to the photo library or transmitted.

09 — Changes to this policy

How we handle updates

If we make a material change to how KIDO handles data — for example, if a future version adds optional cloud backup — we will update this page and increment the "Last updated" date at the top. We will also display a notice in the app at next launch.

We will never introduce data collection that contradicts the core commitment at the top of this page without explicit, separate opt-in consent from a parent or guardian.
