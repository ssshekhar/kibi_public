Legal & Privacy

KIBI™ Privacy Policy

Last updated: 22 July 2026 Applies to: KIBI Android app (all versions) Contact: ssshekharsamal@gmail.com

Core commitment

KIBI is built to work entirely on your phone. Your child's conversations, memories, name, and preferences never leave your device. There is no account, no cloud sync, and no server that receives personal information.

01 — About this policy

Who this applies to

This policy covers the KIBI companion robot Android application published by Shekhar Samal. It describes what information the app handles, where that information lives, and what choices you have as a parent or guardian.

KIBI is intended for use by children under parental supervision. We treat all users as if they may be children and apply the same privacy standards across the board.

02 — What KIBI stores

Everything stays on your phone

KIBI stores the following information locally in your phone's private storage. Nothing in this list is transmitted, backed up to a cloud service, or visible to us.

What Where it lives Who can see it
Child's name & profile On-device database Device only
Conversations with KIBI On-device database Device only
Learned preferences & memories On-device database Device only
Face recognition data On-device storage Device only
Photo Memories (optional) On-device private app storage — off by default, a parent must enable it Device only
Parent settings & PIN On-device encrypted storage Device only
Diagnostic logs On-device file (size-capped, oldest entries rotated out) Device only

If a child or parent flags one of KIBI's generated answers as unsafe or incorrect, the flagged answer is stored on the device for parent review. Nothing is sent to the developer automatically. A parent can choose to send a flagged answer to the developer — see "Parent-confirmed answer reports" in section 03 for exactly what such a report contains.

03 — Internet connections

The only times KIBI contacts the internet

KIBI operates offline by design, and optional "Web answers" are switched off by default — nothing below happens unless a parent turns the feature on in Settings. When enabled, there are three narrow situations where KIBI makes an outbound request, and none of them involves personal information.

Situation Service contacted What is sent What is NOT sent
Child asks a factual question DuckDuckGo (safe search) & Wikipedia Network A short, sanitized subject keyword (a few words at most), never the full sentence Audio, the child's full sentence, name, age, location, device ID, or any personal detail
Child asks about the weather wttr.in (no-account weather API) Network A city name, and only when one is explicitly spoken — weather questions without a city are answered offline, with no location lookup Name, GPS or IP-based location lookup, device ID
A learning card shows a real photo Wikipedia (thumbnail lookup) Network The lesson's subject word only — an animal, planet, landmark, country, or famous-figure name from KIBI's built-in learning content. The image is cached on the device so repeat lessons work offline Audio, the child's words, name, age, location, device ID, or any personal detail

Questions that contain personal language, contact details, or links are kept offline and never sent. Neither service receives an account identifier, device ID, or any information that could be linked back to your child. As with any internet request, the contacted service necessarily sees ordinary connection data such as your IP address; DuckDuckGo operates under its own strict no-tracking policy (safe search parameter kp=1 is always set), and wttr.in is a privacy-respecting open-source weather service that does not build user profiles.

If you leave Web answers off (the default), no such connection ever occurs and KIBI still functions fully: factual questions fall back to the on-device language model, and weather questions get an honest "I'm not sure" response.

Parent-confirmed answer reports. Separately from Web answers, a parent can report a flagged AI answer to the developer. This never happens automatically: the parent opens the PIN-protected dashboard, previews the exact answer, selects a reason, and presses Send. The report contains only three things — the flagged generated answer, the chosen reason, and the app version. It is delivered over HTTPS to a report service running on Cloudflare, which files it as a private GitHub issue that only the developer can read. The report format cannot carry the child's prompt, name, profile, memories, transcript, device identifier, audio, or photos — the app never puts those fields in a report, and the service rejects any payload containing extra fields.

As with any internet request, Cloudflare and GitHub necessarily process ordinary connection data such as the sending IP address in order to deliver the report; it is not stored in the report itself. Their own privacy terms apply to that transport. Each report is sent once, on explicit parent action, with no background retries.

04 — Model downloads

Downloading KIBI's brain on first setup

On first launch, KIBI downloads AI model files (speech recognition, voice, object detection, and language model) from GitHub and public model repositories. These downloads are one-time and can resume if interrupted. While the app is open, KIBI may also check the same public GitHub releases for new offline content packs (stories, quizzes, jokes, facts) and download them the same way.

During download, only the model or content file is fetched. No personal information, device identifiers, or usage data are sent. Once installed, these models and packs run entirely offline.

Model files are stored in your phone's private app storage and are removed when you uninstall the app.

05 — What we never do

Hard limits, not policy choices

The following are architectural constraints, not just policy statements. The app does not contain the code to do any of these things.

  • Sell, share, or license your child's data to any third party
  • Display advertisements of any kind
  • Create a user account or require registration
  • Upload conversations, memories, or voice recordings to a server
  • Use analytics SDKs, crash-reporting services, or behavioural tracking
  • Access your device's contacts, calendar, SMS, or call history
  • Track your location via GPS
  • Send push notifications without an explicit on-device trigger
  • Run in the background when the app is closed

06 — Your rights & controls

You own everything KIBI stores

Because all data lives on your device, you have complete control without needing to contact us.

  • Export your family data From the Parent Dashboard → Family data → Export. Saves a complete ZIP archive (profiles, memories, conversation history, and any Photo Memories) to a location you choose on your device. You move the file yourself — it is never uploaded anywhere.
  • Control Photo Memories Off by default. If you enable it, special-moment photos are saved only to KIBI's private app storage on this device; you can view and delete them from the Parent Dashboard, and disabling the feature stops new captures.
  • Delete a family member's data From the Parent Dashboard → Profiles → select member → Remove. Permanently erases all memories, conversations, and face recognition data for that person.
  • Delete everything Uninstalling the app removes all stored data permanently. Android's standard app data clear (Settings → Apps → KIBI → Clear Data) also works.
  • Disable web lookup Settings → Knowledge → disable "Web answers" and "Weather". KIBI operates fully offline from that point.
  • View diagnostic logs Settings → Advanced → View Diagnostics. Logs are timestamped technical events used to diagnose problems; to make that possible they can include short snippets of what was said and answered. They are size-capped, stay on the device, and are never transmitted — sharing them with anyone is always a manual parent action.

07 — Children's privacy

COPPA and children under 13

KIBI is designed specifically for young children and complies with the U.S. Children's Online Privacy Protection Act (COPPA) and substantially equivalent regulations including the EU's GDPR-K provisions.

We do not knowingly collect personal information from children online. All data described in this policy is stored locally on the family's own device. No information is transmitted to us or to third parties in a form that could identify a child.

The Parent Dashboard is PIN-protected and is the only place where family profiles can be created, edited, or deleted. We encourage parents to review KIBI's settings before handing the device to a child.

If you believe a child's information has been handled improperly, please contact us at ssshekharsamal@gmail.com and we will respond within 72 hours.

08 — Camera & microphone

How KIBI uses device sensors

Microphone: Used only to hear spoken questions and commands while KIBI's listen window is active (triggered by wake word, screen tap, or play mode). Audio is processed entirely on-device by the speech recognition engine. No audio is ever recorded to a file or transmitted.

Camera: Used to detect faces (for profile recognition and presence detection) and objects (for play modes like Treasure Hunt). Live camera frames are processed in memory at approximately 1–2 frames per second and immediately discarded — nothing is transmitted. If a parent enables the optional Photo Memories feature (off by default), KIBI saves occasional special-moment photos to its private app storage on this device only; they never appear in the system photo library, are never uploaded, and can be viewed and deleted from the Parent Dashboard.

Bluetooth robot (optional): Pairing with the KIBI robot chassis uses Android's Companion Device Manager: Android itself performs the discovery and shows a system chooser that a parent must approve. KIBI does not scan for other devices and does not request device location.

09 — Changes to this policy

How we handle updates

If we make a material change to how KIBI handles data — for example, if a future version adds optional cloud backup — we will update this page and increment the "Last updated" date at the top. We will also display a notice in the app at next launch.

We will never introduce data collection that contradicts the core commitment at the top of this page without explicit, separate opt-in consent from a parent or guardian.
